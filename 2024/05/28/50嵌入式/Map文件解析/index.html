Map文件解析

ARM map文件

.map 文件对应的中文名应该是映射文件，用来展示（映射）项目构建的链接阶段的细节。通常包含程序的全局符号、交叉引用和内存映射等等信息。目前，常见的编译器套件（实际是其中的链接器），例如， GCC、ARMCC、IAR 都可以生成 .map 文件

基础数据类型

程序编译完后会有三种数据（Data）段

RO：Read only只读数据，指令（Code）和常量

RW：Read/write可读可写数据，已初始化变量

ZI：Zero Init数据,未初始化,或者用0初始化的变量

Totil RO Size (Code + RO Data) 包含指令和常量

Totil RW Size(RW Data+ ZI Data) 已经初始化和为未初始化的数据

Totil ROM Size(Code + RO Data+ RW Data)

ARM image

指烧录到ROM中的bin/Hex文件,也称为image文件

Image文件包含了RO和RW数据,之所以Image文件不包含ZI数据,是因为ZI数据都是0,只要程序运行之前将ZI数据所在的区域一律清零即可

在image的指令中，至少需要

  1. 将RW从ROM中搬到RAM中
  2. 将ZI所在的RAM区域全部清零

ARM程序的空间属性

bss段: BSS段（bss segment）（Block Started by Symbol）通常是指用来存放程序中未初始化的全局变量的一块内存区域.,BSS段属于静态内存分配.

data段: 数据段（data segment）通常是指用来存放程序中已初始化的全局变量的一块内存区域,数据段属于静态内存分配.

text段：代码段（code segment/text segment）通常是指用来存放程序执行代码的一块内存区域,这部分区域的大小在程序运行前就已经确定,通常只读,某些架构也允许代码段为可写,即允许修改程序.在代码段中,也有可能包含一些只读的常数变量,例如字符串常量等

map文件分析

.map文件时编译器链接时生成的一个文件，它主要包含了交叉链接信息。通过.map文件，我们可以知道整个工程的函数调用关系、FLASH和RAM 占用情况及其详细的汇总信息。能具体到单个源文件（.c/.s）的占用情况，根据这些信息，我们可对代码进行优化。

GCC中.map文件

  1. 存档文件引用关系
  1. 丢弃的输入段
  1. 内存配置
  1. 链接脚本和内存映射

ARMCC中.map文件

.map文件可以分为五个部分:

  1. 程序段交叉引用关系（Section Cross References）
  2. 删除映像未使用的程序段（Removing Unused input sections from the image）
  3. 映像符号表（Image Symbol Table ）
  4. 映像内存分布图（Memory Map of the image）
  5. 映像组件大小（Image component sizes）
