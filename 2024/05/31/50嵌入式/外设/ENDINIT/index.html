ENDINIT

简介

SCU内置endinit watchdog，用于控制系统寄存器访问保护，被endinit保护的寄存器写操作需要先开启endinit watchdog，并在watchdog设定的时间范围内完成配置，之后重新锁定endini，否则会输出报警。（endinit清除需要一定时间，建议在清除操作后进行一次回读，再访问受保护的寄存器）

主要特性

endinit包含如下特性:

  • 在第一次发起passwordaccess后开始工作;
  • WDT功能可以禁止，但timeout功能无法禁止;
  • 支持otp配置禁止endinit功能;
  • 支持debug配置禁止endinit功能;
  • 在password或者modify访问错误时报警;
  • 在喂狗密码错误时产生报警;
  • 喂狗操作支持静态密码和动态密码两种模式;
  • password，check和modify访问支持动态pattern和静态pattern方式;
  • 支持在WDT模式下输出唤醒中断信号;

功能描述

EndIinit Watchdog Timer

endinit watchdog 工作流程

复位后WDG默认关闭，发起password access激活

配置被关闭endinit保护的寄存器的操作流程

  1. 通过password access，超时模式开始工作
  2. modify access, 配置endinit为0，关闭endinit
  3. 配置被endinit保护的寄存器
  4. 完成配置后通过modify access，配置配置endinit为1，开启endinit，此时WDG自动退出超时模式（配置配置endinit为0的持续时间由WDTSCON1配置，如果超时后仍没有打开endinit，将输出报警，endinit的超时模式在password access激活后无法禁用）

watchdog mode 功能:

endinit watchdog 可以用做 watchdog 功能，可以产生定时唤醒中断和输出 watchdog 超时报警，endinit 从 timeout mode 退出后会自动进入 watchdog mode,watchdog mode 的计数相关参数可以配置由 endinit 保护的 WDTSCON1 寄存器设置，也可以设置 WDTSCON1.DS=1，禁用 endinit 的watchdog 功能。
在 endinit watchdog 有效时，可以通过 password access 和 modify access 组合喂狗，也可以通过 check access 修改喂狗的密码。endinit watchdog 在Lck后，如果 pas=1，则需要使用动态密码喂狗，且每次 password access和 check access 均会修改密码，如果 pas=0，则使用 pw 中静态密码喂狗。

watchdog mode 低功耗周期唤醒功能

在 watchdog mode 下，endinit watchdog 内部计数器在计数到 0x8000 时会发起中断给 cpu，该功能默认不使能，可以通过配置 WDTSCON1.IREN 打开

EndIinit Timer

为了防止 endinit watchdog 寄存器保护功能异常导致寄存器无法配置的单点失效情况，内部设置了endinit timer，该模块不支持 watchdog mode，仅用于保护寄存器访问，寄存器保护和 endinitwatchdog 类似，会输出endinit用于控制寄存器的访问，如下图:

endinit watchdog 和 endinit timer 任意一个 endinit 关闭时，均可以配置寄存器，同时 debug 可以配置关闭 endinit 功能。
复位后 endinit timer是 disabled 的，在第一次 password access 后激活，激活后不可以关闭。

操作模式

endinitwatchdog有三种操作模式

只有当LCK位置起时，才能进行password 或者 check access

  • password access

为了保证正确的服务顺序，只有LCK位置起时才允许密码访问，密码访问会解锁LCK位，之后可以进行修改访问，以对endinit进行配置，若访问错误则立即产生告警

bit位 要求的值
1:0 固定值，必须写入2`b01
15:2 如果PAS=0
WDTSCON0.PW[7:2]需要写入从WDTSCON0.PW[7:2]读出数据的翻转值WDTSCON0.PW[15:8]必须写入从WDTSCON0.PW[15:8]读取的数据
如果PAS=1
必须写入下一个密码序列
31:16 如果TCS=0
必须写入定时器重载值WDTxCON0.REL
如果TCS=1
需要写入当前计数值(WDTxSR.TIM)的估计值，误差需要在±容忍值（WDTxSR.TCT）以内
  • check access

只有LCK位置起时才允许检查访问

bit位 要求的值
1:0 固定值，必须写入2`b11
15:2 如果PAS=0
WDTSCON0.PW[7:2]需要写入从WDTSCON0.PW[7:2]读出数据的翻转值
WDTSCON0.PW[15:8]必须写入从WDTSCON0.PW[15:8]读取的数据
如果PAS==1
必须写入下一个密码序列
31:16 如果TCS=0
必须写入定时器重载值WDTxCON0.REL
如果TCS==1
需要写入当前计数值(WDTxSR.TIM)的估计值，误差需要在±容忍值（WDTxSR.TCT）以内
  • modify access
bit位 要求的值
0 用户定义，期望设置的WDTSCON0.ENDINIT的值
1 固定值1
15:2 用户定义，期望设置的WDTSCON0.PW的值
31:16 用户定义，期望设置的WDTxCON0.REL的值

访问受Endinit保护的寄存器

如果在运行期间需要对受 Endinit 保护的寄存器进行写入访问，则可以在有限的时间内临时重新启用写入访问。 提供了两种选择：

  1. 通过WDT启用对受ENDINIT保护的寄存器的访问

    1. 使用有效的密码访问解锁WDTxCON0
    2. 使用有效的修改访问清除ENDINIT
    3. 访问被保护的寄存器

    注意：当WDTxCON0解锁时，WDT自动切换到超时模式，因此访问窗口有时间限制，超时模式在重新设置ENDINIT后终止。

  2. 不通过WDT对ENDINIT保护的寄存器的访问

    直接禁用WDT(WDTxSR.DS=1)(不建议)

WDTSCON0

Bitfield Name ResetVal Access Desc
31:16 REL 0xFFFC RW WDT的复位值（同时也是时间检查值）
该值可以在一次WDTxCON0的modify access中被修改
如果WDT开启，并且在NormalTimerMode，在喂狗后，从该值持续递减
在password access或者check access期间，此位域被用于额外的检查，在检查期间写入不会影响复位值
如果WDTxSR.TCS=0，在password access或者check access时，该位域必须写入当前值
如果WDTxSR.TCS=1，在password access或者check access时，该位域必须写入WDTxSR.TIM的值，允许误差在一定范围内
15:2 PW 60 RW 可用户自定义的密码以访问WDTxCON0
当modify access时此位域必须写入初始密码
读取该位域会返回初始密码（TODO）
1 LCK 1 RW 控制访问WDTSCON0的锁定位
由硬件控制
当WDTSSR.US为0时，正确的password access会清除锁定位，当一次正确的modify accessWDTSCON0后，锁定位会自动置起
在写入WDTSCON0期间，对该位的写入仅用于密码保护机制，不会实际储存
该位一定会在password access后清除，在modify access后置起
check access不会清除此位
0表示不上锁，1表示上锁
0 ENDINIT 0 RW endinit控制位
当进行密码或者check access时，该位只能写1（即使这次写入仅仅用于密码保护机制并且不会保存），在modify access时该位只能写入需要的值
该位用于访问带E的寄存器，但是当WDT失效时，候补寄存器SEICON0.ENDINIT可以达到相同效果
0：允许访问（当应用复位时）
1：不允许访问

WDTSCON1

Bitfield Name ResetVal Access Desc
31:16 保留 0 RO 读返回0需要写0
15:9 TCTR 0 RW 计时器检查容差请求
只能在endinit关闭时修改
WDTxSR.TCT会更新当ENDINIT开启
只要ENDINIT、关闭，WDTxSR.TCT控制计时器检查的容差
当endinit重新启动，WDTxSR.TCT更新为TCTR的状态
8 TCR 0 RW 计数器检查请求位
WDTxSR.TCS更新当ENDINIT重新开启
当ENDINIT关闭，WDTxSR.TCS控制计数器检查是否开启当ENDINIT重新开启，WDTxSR.TCS更新为TCR的状态
0：只检查REL被写入存在的REL值在modify access或者check access
1：检查REL被写入的值为TIM值（允许容差范围内）在modify access或者check access
7 PAR 0 RW 密码自动序列请求位
可以被修改当ENDINIT关闭
只要ENDINIT关闭，WSTxSR.PAS控制密码序列，WSTxSR.PAS会更新为PAR的状态
0：不自动更新密码在每次修改或者check access之后
1：自动更新密码在每次修改或者check access之后
6 IREN 0 RW 中断开启控制位
0：关闭中断
1：开启中断当，WDTx计数器到0x8000
5 IR1 0 RW 输入频率请求控制位
位IR0和IR1应该一起被编程来探测WDTx计数器的频率
只要ENDINIT关闭，WSTxSR.IS0和WSTxSR.IS1控制当前输入频率对WDT，当ENDINIT开启，WSTxSR.IS0和WSTxSR.IS1更新为IR0和IR1
0
IR0=0fSPB/16384
IR0=1fSPB/256
1
IR0=0fSPB/64
IR0=1保留
4 保留 0 RO
3 DR 0 RW 关闭请求控制位
0请求开启WDTx
1请求关闭WDTx
2 IR0 0 RW
1 保留 0 RO
0 CLRIRF 0 RW 重启内部复位标志

WDTSSR

Bitfield Name ResetVal Access Desc
31:16 TIM 65532 RO 定时器值
反应WDTx当前计数值
15:9 TCT 0 RO 定时器检查容差值
此位域指示password access或者check access的容差值，当WDTxCON0.ENDINIT在一次正确的modify access被写1时，此位域会被WDTxCON1.TCTR的值更新
8 TCS 0 RO 定时器检查状态标识
当WDTxCON0.ENDINIT在一次正确的modify access被写1时，此位域会被WDTxCON1.TCR的值更新
0：当修改或者check access时REL需要写入正确的REL值
1：当修改或者check access时REL需要写入当前TIM值允许误差范围±TCT
7 PAS 0 RO 自动密码序列状态标志位
当WDTxCON0.ENDINIT在一次正确的modify access被写1时，此位域会被WDTxCON1.PAR的值更新
0：在每次修改或者check access后不修改密码
1：在每次修改或者check access后自动修改密码
6 IRS 0 RO 中断状态标志位
当WDTxCON0.IREN置1并且WDTx的计数值到达0x8000时此位置1
当WDTxCON0.ENDINIT在一次正确的modify access被写1时，此位清除
5 IS1 0 RO 看门狗输入时钟状态
IS0和IS1应该被一起修改，它们用来指示当前WDTx的时钟频率（规则见上文）
当WDTxCON0.ENDINIT在一次正确的modify access被写1时，这两个位会被WDTxCON0.IR0和WDTxCON0.IR1修改
4 TO 1 RO 看门狗超时模式标志位
当进入超时模式时此位被置起，当退出超时模式时，此位被清除
0：看门狗不在超时模式下
1：看门狗在超时模式下
3 DS 0 RO 看门狗开启/关闭状态标志位
当WDTxCON0.ENDINIT在一次正确的modify access被写1时，此位被更新为WDTxCON1.DR的值
当进入超时模式时，此位被清除
0：WDTx开启
1：WDTx关闭
2 IS0 0 RO 看门狗输入时钟状态，见IS1
1 OE 0 RO 看门狗溢出错误状态位
当WDTx从0xFFFFH计数到0x00000时，此位置起
当WDTxCON0.ENDINIT在一次正确的modify access被写1时此位清除
0：无看门狗溢出错误
1：一次看门狗溢出错误发生
0 AE 0 RO 看门狗访问错误状态标志位
当一次错误password access或者modify access发生时，此位置起
当WDTxCON0.ENDINIT在一次正确的modify access被写1时此位清除
0：无看门狗访问错误
1：一次看门狗访问错误发生
